# 基本指令與觀念


提交訊息是查看其他人提交的修改內容或自己檢查歷史記錄時重要的資料。
所以要用心填寫讓人容易理解的提交訊息。

Git 的標準提交 (commit) 訊息：

第 1 行：提交時修改內容的摘要
第 2 行：空行
第 3 行以後：修改的理由

建議以這種形式填寫提交訊息，詳細可以參考 Git commit specification
